# Project name: club's platform

## 1. 개요 (Overview)
- 동아리들이 사용해야 할 플랫폼이 너무 많음(노션, 단톡, 디스코드 등).
- 카톡으로 공지를 할 수 있지만, 노션 등에 올리는 게시물이나 공지 등이 휴대폰 알람으로 이어지지 않아 확인하지 않는 경우가 많음.
- 동아리에서 플랫폼을 구축하기 위한 시간 소요가 너무 많다고 생각함 -> 동아리 운영에 필요한 기능을 제공하는 플랫폼이 필요.
<br>

-   There are too many platforms that clubs have to use (Notion, group chats, Discord, etc.).
-   While it's possible to make announcements via KakaoTalk, posts and notices on platforms like Notion do not trigger mobile notifications, often leading to them being ignored.
-   There is a significant time investment required to build a platform for club management. A platform that provides the essential functions needed for managing clubs is necessary.

### 1.1 기술 스택 (Skills)
- 프론트엔드 (Frontend): React (Web), React Native (Mobile)
- 백엔드 (Backend): Java (with Spring Boot)
- 데이터베이스 (Database): Firebase (for real-time data), MySQL (for relational data)
- 실시간 통신 (Real-time Communication): WebSocket
- 인증 (Authentication): OAuth2, JWT


## 2. 사용자 웹/앱 핵심 요구사항

### 2.1 기능적 요구사항

1. 회원가입

- **사용자 앱/웹**
    - Google 로그인
    - 프로필 설정: 실명, 분야, 프로필 이미지 등 기본 정보 입력
    - 이용약관 및 개인정보 처리방침 동의 절차

2. 커뮤니티 게시판

- 게시글 작성, 조회, 수정, 삭제(CRUD) 기능 제공
- 댓글 및 대댓글 기능 포함
- 좋아요 및 북마크 기능 지원
- 게시판 카테고리화(공지사항, 자유 게시판 등)
- 게시글 검색 및 필터링 기능
- 첨부파일 업로드 기능(이미지, 문서 등)

3. 알림 시스템

- 관리자 및 사용자가 주요 활동에 대해 알림을 받을 수 있도록 구현
- 이메일 알림 및 푸시 알림 제공
- 알림 설정 기능: 사용자별 알림 유형 및 빈도 설정 가능
- 읽음/안읽음 표시 및 알림 기록 관리

4. 관리자 페이지

- 회원 관리: 가입 승인, 회원 삭제, 역할 부여(일반 사용자, 관리자)
- 질문지 업데이트: 면접 질문의 등록/수정/삭제 가능
- 통계 기능: 회원 가입, 게시글 작성 등 주요 활동 통계 제공

5. 프로필 페이지

- 사용자 프로필 관리 기능: 사진 업로드, 정보 수정
- 추가정보 입력: 실명, 분야
- 활동 내역 조회 기능(작성한 게시글, 댓글 등)
- 관심 카테고리 설정 및 맞춤 콘텐츠 제공 기능

6. 파트너 동아리 등록 및 관리

- 파트너 동아리 신청 기능:
    - 동아리명, 회장정보, 연락처, 이메일, 소개글 등록(마크다운 지원)
    - 동아리 로고 및 배너 이미지와 소개 PPT 등의 파일 업로드
    - 동아리 회원 목록 관리
- 파트너 동아리 목록 조회 및 상세 정보 확인
- 파트너 동아리 활동 내역 공유(게시글, 이벤트 등)
- 동아리 회원 인증 및 가입 신청 기능

7. 채팅 시스템

- 1:1 개인 채팅 기능
- 파트너 동아리 그룹 채팅방
- 메시지 저장 및 검색 기능
- 읽음 확인 및 알림 기능
- 파일 및 이미지 공유 기능
- 채팅방 관리 기능(초대, 나가기, 설정 등)

### 2.2 사용자 웹/앱 비기능적 요구사항

1. 성능 요구사항

- 동시 접속자 수 최대 100명 기준 성능 최적화
- 평균 응답 시간 1초 이내 유지
- 이미지 및 첨부파일 로딩 시간 최적화(이미지 압축, 점진적 로딩)
- 모바일 환경에서 데이터 사용량 최소화

2. 보안 요구사항

- OAuth2 및 JWT(Json Web Token)를 활용한 인증 및 권한 관리
- 데이터 암호화: 비밀번호는 BCrypt로 암호화, 주요 데이터 HTTPS로 통신
- CSRF/XSS 방어 및 SQL Injection 방지 구현
- 파일 업로드 보안: 허용된 파일 형식 및 크기 제한, 악성코드 검사

3. 확장성 요구사항

- 커뮤니티 확장 가능성을 고려한 모듈화된 설계
- 향후 동시 사용자 증가를 대비한 수평 확장(서버 증설) 가능
- 다국어 지원을 위한 국제화(i18n) 구조 적용

4. 유지보수성 요구사항

- 명확한 코드 구조 및 주석 작성
- 자동화된 테스트 스크립트 포함
- 에러 로깅 및 알림 시스템 구축
- 사용자 피드백 수집 및 개선 프로세스 구축

5. 사용성 요구사항

- 직관적인 UI/UX 설계
- 모바일 및 데스크톱 환경 모두 최적화된 반응형 디자인
- 접근성 고려: 스크린 리더 지원, 키보드 탐색 등
- 초기 사용자 온보딩 및 도움말 기능

## 3. 관리자 웹/앱 핵심 요구사항

### 3.1 기능적 요구사항

1. 회원관리
    - Google API를 통해 가져 올 회원정보: 이름, 프로필 이미지 URL, 이메일 주소
    - 회원관리: 회원 삭제, 이메일 보내기, 1:1 채팅
    - 회원 권한 관리: 일반 사용자, 관리자 역할 부여 및 변경
    - 회원 활동 조회: 게시글, 댓글, 신고 이력 등 확인
    - 회원 활동 제한: 임시 차단, 영구 차단, 경고 등 제재 조치
    - 회원 통계: 활동 패턴, 참여도, 기여도 등 분석
2. 파트너 동아리 관리
    - 파트너 동아리 신청 검토 및 승인/거부 프로세스
    - 파트너 동아리 정보 관리 및 수정
    - 파트너 동아리 채팅방: [파트너 동아리 신청]에서 '수락'하면 신청한 회원이 초대됨
    - 파트너 동아리 활동 모니터링 및 지원 기능
    - 파트너 동아리 공지사항 및 이벤트 관리
    - 파트너십 등급 및 혜택 관리 시스템
3. 카테고리 관리
    - 카테고리 편집(최대 20개)
    - 3단계 카테고리 구조(상-중-하)
        - 예시
            - 상(개발) - 중(Frontend) - 하(HTML/CSS)
            - 상(개발) - 중(Frontend) - 하(JavaScript)
            - 상(개발) - 중(Frontend) - 하(TypeScript)
            - …
    - 카테고리별 게시판 설정: 공개/비공개, 작성 권한 설정
    - 카테고리 사용 통계 및 활성화/비활성화 관리
4. 콘텐츠 관리
    - 게시글 및 댓글 관리: 부적절한 콘텐츠 필터링, 수정, 삭제
    - 신고된 콘텐츠 처리: 신고 내역 확인, 콘텐츠 검토, 조치 적용
    - 공지사항 관리: 전체/카테고리별 공지 작성, 수정, 삭제, 상단 고정
    - 콘텐츠 품질 관리: 우수 콘텐츠 선정 및 강조 기능
    - 콘텐츠 자동 필터링 규칙 설정 및 관리
5. 통계 및 분석
    - 대시보드: 일간/주간/월간 활동 통계, 신규 가입자 수, 활성 사용자 수 등
    - 인기 콘텐츠 분석: 조회수, 댓글, 좋아요 기준 인기 게시글 집계
    - 카테고리별 활동 통계: 카테고리별 게시글 및 사용자 참여도 분석
    - 사용자 행동 패턴 분석: 방문 시간, 체류 시간, 주요 활동 분석
    - 성장 지표 모니터링: 회원 증가율, 활동량 변화 등
    - 통계 데이터 내보내기 기능(CSV, Excel 등)
6. 시스템 설정
    - 알림 설정: 관리자 알림 규칙 설정, 일괄 알림 발송
    - 금지어 관리: 자동 필터링될 단어 설정 및 관리
    - 백업 및 복원: 데이터 백업 일정 설정, 백업 데이터 복원
    - 시스템 로그 조회: 오류 로그, 접속 로그, 주요 활동 로그 확인
    - 시스템 성능 모니터링 및 최적화 도구
    - API 토큰 관리 및 접근 제어 설정
7. 채팅 관리 시스템
    - 관리자 채팅방 모니터링 및 중재 기능
    - 불량 메시지 필터링 및 신고 처리
    - 채팅 로그 저장 및 분석
    - 중요 공지 및 시스템 메시지 전체 발송 기능

### 3.2 비기능적 요구사항

1. 성능 요구사항
    - 관리자 대시보드 로딩 시간 2초 이내
    - 대량 데이터 처리 시(회원 관리, 통계 분석) 페이지네이션 적용
    - 실시간 모니터링 데이터 업데이트 주기 5분 이내
    - 대량 작업(일괄 메일 발송, 데이터 내보내기) 백그라운드 처리
2. 보안 요구사항
    - 관리자 계정 2단계 인증(2FA) 의무화
    - 관리자 접속 IP 제한 기능 지원
    - 관리자 활동 로그 자동 기록 및 보관(최소 6개월)
    - 권한 수준에 따른 기능 접근 제어 적용
    - 정기적인 보안 감사 및 취약점 점검 프로세스
    - 관리자 계정 패스워드 정책 강화(복잡성, 주기적 변경)
3. 사용성 요구사항
    - 직관적인 관리자 UI/UX 설계
    - 모바일 환경에서도 주요 관리 기능 사용 가능(반응형 디자인)
    - 관리자 작업 절차 최소화(클릭 수 최소화)
    - 데이터 시각화: 그래프, 차트 등을 활용한 통계 데이터 표현
    - 관리자 대시보드 사용자 정의 기능
    - 빠른 검색 및 필터링 기능
4. 확장성 요구사항
    - 여러 관리자가 동시에 시스템 사용 시 충돌 방지 기능
    - 향후 기능 확장을 고려한 모듈식 설계
    - API 기반 구조로 외부 시스템 연동 가능성 확보
    - 관리자 권한 레벨 세분화 지원
5. 유지보수성 요구사항
    - 관리자 기능 변경/추가 시 최소한의 코드 수정으로 구현 가능한 구조
    - 관리자용 매뉴얼 및 도움말 시스템 구축
    - 오류 발생 시 명확한 메시지와 해결 방법 제시
    - 자동화된 시스템 상태 모니터링 및 알림
